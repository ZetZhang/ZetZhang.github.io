Docker基本使用

Posted on 2020-06-12 | In 2020.06

修改镜像源

（manjaro）只需要添加配置文件/etc/docker/daemon.json

或者使用–registry-mirror选项

  • 修改:/etc/default/docker
  • 添加:DOCKER_OPTS=”–registry-mirror=https://docker.mirrors.ustc.edu.cn"
  • systemctl daemon-reload
  • systemctl restart docker

一些Docker基础命令

容器操作

run: 通过运行镜像输出hello

docker run CONTAINER [COMMAND] [ARGS]

create: 创建容器

start: 启动容器

ps: 查看容器

docker ps [OPT]

rm: 删除容器

docker rm '{IMAGE}'

attach: 继续运行后台守护容器

docker attach '{IMAGE}'
dokcer attach [--detach-keys[=[]]] [--no-stdin][--sig-proxy[=true]] CONTAINER

exec: 在已运行的容器中启动新进程

docker exec [OPT] CONTAINER [COMMAND] {ARGS}

stop/kill: 停止容器

docker stop [-t | --time[=10]] CONTAINER
docker kill CONTAINER

镜像操作

images/inspect: 查看镜像

docker iamges [OPT] [REPOSITORY]
docker inspect [OPT] CONTAINER|IMAGE [...]

search: 查找镜像

docker search [OPT] TERM

pull: 拉取镜像

docker pull [OPT] NAME [:TAG]

构建镜像

commit: 通过容器

docker commit [OPT] CONTAINER [REPOSITORY[:TAG]]

import: 通过本地模板导入

docker import [OPT] file |URL|[REPOSITORY[:TAG]]

build: 通过Dockerfile文件

docker build

通过Dockerfile

Dockerfile格式：
Dockerfile构建过程
  • 从基础镜像运行容器
  • 执行指令，对容器做修改
  • 执行类docker commit操作，提交新镜像层
  • 基于新镜像运行新容器
  • 执行Dockerfile的下一指令直到所有指令执行完毕

save: 存入镜像

docker save

load: 载入镜像

docker load

push: 上传镜像

docker push NAME[: TAG] |REGISTRY_HOST[: REGISTRY_PORT]/NAME[: TAG]

rmi: 删除镜像

docker rmi [OPT] IMAGE [...]
